Security

Authentication

Markland uses magic-link email for human sign-in and scoped bearer tokens for MCP clients. Tokens are issued per-user and per-agent; you can revoke them individually from the settings page.

Data handling

Documents are stored in SQLite on the server. Grants are per-document; there is no “anyone with the link” default. All traffic is HTTPS.

Reporting issues

Found a security issue? Email security@markland.dev. Please allow a reasonable window before public disclosure.
