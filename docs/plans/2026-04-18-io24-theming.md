# IO24 Theming Pass — Gemini Gradient Aesthetic (Superseded)

**Date:** 2026-04-18 mid-day
**Status:** **Superseded** — replaced later the same day by the Dark-Outlined-Primary pass. See `2026-04-18-dark-outlined-primary.md`.
**Reference:** https://io.google/2024/ (light-mode, gradient-heavy interpretation)
**Supersedes:** `2026-04-18-neubrutalism-theming.md`

## Why This Was Superseded

A design review flagged the gradient pass as too soft and too gradient-heavy — reading as "indie SaaS editorial" rather than the "exhibition/stage-set" energy the io.google/2024 page actually projects. The user pointed specifically to the dark two-up outlined-tile pattern from the I/O page ("Join a community group" / "Continue learning") as the intended reference. This retrospective is retained for history; the current visual authority is the dark-outlined-primary retrospective.

Key failures of this pass:
- Too much gradient surface — hero blobs, rainbow text, top strips on cards, gradient backdrop mesh. Cumulatively it read as "AI slop."
- Light canvas made the page feel friendly but not cool/modern.
- Primary colors were used as gradient STOPS, never as discrete solid accents — so they never punched.
- Too many decorative elements (sparkle icons across eyebrows, hero chip, CTA, search button, section eyebrows).

## Why the Pivot

The neubrutalism pass shipped in the morning landed technically but didn't match what the user actually wanted on review. When the user pointed to the Google I/O 2024 landing page and said "this is exactly the style I want," the direction inverted:

| Dimension | Neubrutalism (AM) | IO24 (PM) |
|-----------|-------------------|-----------|
| Corner language | Hard edges, 0–4px radius | Rounded, 20–28px cards, pill buttons/inputs |
| Shadow style | Solid offset `5px 5px 0 0 #000` | Soft blur `0 10px 30px rgba(...)` |
| Color energy | High-contrast primaries against cream | Soft gradient mesh + saturated signature rainbow |
| Typography | Archivo Black UPPERCASE + JetBrains Mono | Figtree 400–900 + DM Mono (sparingly) |
| Motion | Static confidence, press-in/out only | Sticky blur nav, gradient text animation, lift-on-hover |
| Tone | Raw, loud, "sticker" | Optimistic, tech-forward, Gemini-era Google |

Both are legitimate aesthetic commitments; the user picked the second.

## Aesthetic Summary

Warm off-white `#FAFAF7` canvas with an ambient three-stop radial gradient mesh (blue + magenta + yellow bleeding in from edges — subtle, ~8–12% opacity). Signature rainbow gradient (`blue → purple → magenta → orange → yellow`) applied to the wordmark and hero key words, card top strips, and the `.sparkle` icon fill. Rounded forms everywhere. Typography is Figtree (open-source Google-Sans analogue), with Newsreader serif retained for document-reading contrast and DM Mono for metadata.

## Key Design Decisions

### 1. Figtree over Archivo Black

Archivo Black is a beautiful face but it has a single weight (900) and reads as brutalist-signature. IO24 lives on weight contrast — Google Sans moves from 400 through 800 in the same family. Figtree is the closest free alternative: geometric-humanist with the same proportions and a full weight range. Used throughout: body in 400/500, buttons in 600, card titles in 700, hero headlines in 800.

### 2. Gradient placement — sparing, not everywhere

The rainbow gradient is the single most-identifying visual. Overusing it kills the effect. The gradient is used ONLY on:

1. Wordmark (text + dot)
2. Hero headline second line ("The world reads.")
3. Hero chip sparkle icon + CTA sparkle icon
4. 4px top strip on the doc container + on explore cards (cycling)
5. Brand dot on doc viewer

Everything else is off-white, white, or muted gray. Restraint is what makes the gradient punch.

### 3. Keep Newsreader for doc reading

One thing from the neubrutalism pass that didn't need to change: the document viewer uses Newsreader variable serif for body copy. Reading long-form markdown is a different UX problem than landing-page energy. Figtree for everything else, Newsreader only for body text in `/d/<token>`.

### 4. Soft shadows + pill buttons signal "SaaS-premium" in 2026

In the current design-language landscape, soft drop shadows with high border-radius read as "modern premium AI app" (Linear, Arc, Raycast, Vercel, OpenAI). Brutalism with hard offset shadows reads as "indie-developer blog" (which has its own audience). The IO24 aesthetic positions Markland as the former.

### 5. Sparkle as a recurring motif

The four-pointed star/sparkle is the Gemini-era Google accent. Implemented here as a reusable `.sparkle` class — CSS `mask` pointing to an inline SVG path, filled with the `--gradient-rainbow`. Sized via `width/height` so it can scale inline with text. Used in the hero chip, CTA icon, section eyebrows, search button — repeated enough to be a motif, not so often that it dilutes.

### 6. Ambient gradient mesh, not gradient blobs on every surface

Body background gets a soft three-stop radial gradient mesh so the page never reads as flat. This is different from neubrutalism's polka-dot grain (which created a paper texture). The mesh creates atmosphere without competing with content — eyes register it peripherally.

### 7. Motion: restrained but present

The headline gradient animates its `background-position` on a 10s loop — barely perceptible color drift. The wordmark gradient shifts on hover. Cards and buttons translate `-2px`/`-4px` on hover with shadow growth. No scroll animations, no parallax, no loading reveals. Enough motion to feel alive, not enough to distract.

## What's Next (If Pursued)

- **Dark mode toggle** — io.google/2024 has one. Adding it means parallel dark tokens for `--bg`, `--surface`, and the hairlines, plus a theme toggle in the header.
- **Hero blob animation** — currently static positioned gradient blobs. A slow `@keyframes` drift on their positions would add more Gemini-era motion.
- **SVG "sparkle" particles** on hover over interactive elements (the actual IO24 page has subtle particle effects on CTAs).
- **Variable-weight headline animation** — Figtree is variable-axis capable; headlines could subtly shift weight on view.
- **Doc viewer "reading mode" toggle** — Figtree body vs Newsreader body.
- **Custom cursor on landing** — colored blob cursor that follows mouse.

None of these are required. The current pass delivers the aesthetic.

## Not In Scope

- Dark mode (tokens are structured for it; not wired up)
- Component library extraction
- Accessibility audit (contrast ratios look good by eye; the rainbow-gradient text against off-white warrants a formal check before public launch)
- Logo design beyond the wordmark
