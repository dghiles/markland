# Presence — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

> **Note:** No git commits during execution — this project is not a git repo at time of writing; the user manages version control manually. Where commit steps would normally appear, a "verification" step is substituted.

**Goal:** Add advisory presence signaling to Markland. Users and agents can announce `reading` or `editing` on a document with an optional note; other principals see who is active via `markland_get` and the web viewer. Presence is **non-enforcing** — it does not lock or reject writes. Rows have a 10-minute TTL, refreshed by a ~5-minute client heartbeat (documented, not enforced). Stale rows are filtered in-query and swept by a 60-second background GC task. This is plan 9 of 10 per `docs/specs/2026-04-19-multi-agent-auth-design.md` §17, implementing §9 and the presence portions of §10.1 and §12.2.

**Architecture:** A new `presence` table keyed on `(doc_id, principal_id)`. A `service/presence.py` module provides `set_status`, `clear_status`, `list_active`, and `gc_expired`, each accepting an optional `now` parameter (defaults to `datetime.utcnow`) so tests can inject time without mocking. MCP exposes `markland_set_status` and `markland_clear_status`; `markland_get` is extended to embed `active_principals` (joined against `users` / `agents` for `display_name`). HTTP mirrors with `POST/DELETE/GET /api/docs/{id}/presence`. A background asyncio task registered in `run_app.py`'s FastAPI lifespan calls `gc_expired()` every 60 seconds; the in-query filter (`expires_at > now`) ensures correctness even if GC is slow or crashes. The web viewer at `/d/{share_token}` renders a minimal "X is editing · since N min ago" badge when active principals exist — no SSE, no live updates, page refresh to see changes.

**Tech Stack:** Python 3.12, FastAPI, FastMCP, uvicorn, SQLite (WAL), Jinja2. No new third-party dependencies.

**Scope excluded (this plan):**
- No SSE/WebSocket live updates — web viewer requires refresh to see changes
- No cursors, selection ranges, or per-section presence
- No explicit "done" state — finishing work means `clear_status` or let expire
- No status beyond `reading` / `editing`
- No enforcement — edits succeed regardless of who is marked `editing`
- No per-principal rate limits on `set_status` (covered in plan 10)
- No audit-log entries for presence changes (high-churn, low-value; plan 10 decides)
- No push notification to other principals when someone starts editing
- No presence history / who-was-here-recently surface

```python
"""Advisory presence — who is reading or editing a document.

Non-enforcing: calling `set_status(status='editing')` does NOT prevent other
principals from writing. Conflict safety lives in service/docs via the version
column; presence is a coordination hint only.

Every time-dependent function accepts an optional `now: datetime | None`
parameter. Production code passes nothing; tests pass explicit timestamps to
avoid mocking the clock.
"""

from __future__ import annotations

import sqlite3
from dataclasses import dataclass
from datetime import datetime, timedelta
from typing import Literal

from markland.service.auth import Principal

_TTL = timedelta(minutes=10)

Status = Literal["reading", "editing"]
_VALID_STATUSES: tuple[Status, ...] = ("reading", "editing")


class PresenceError(RuntimeError):
    """Raised when a presence operation fails (e.g. doc not found, no access)."""


@dataclass(frozen=True)
class ActivePrincipal:
    """One row of `list_active` output."""
    principal_id: str
    principal_type: str
    display_name: str | None
    status: str
    note: str | None
    updated_at: str


def _now(now: datetime | None) -> datetime:
    return now if now is not None else datetime.utcnow()


def _doc_exists(conn: sqlite3.Connection, doc_id: str) -> bool:
    row = conn.execute("SELECT 1 FROM documents WHERE id = ?", (doc_id,)).fetchone()
    return row is not None


def set_status(
    conn: sqlite3.Connection,
    *,
    doc_id: str,
    principal: Principal,
    status: Status,
    note: str | None = None,
    now: datetime | None = None,
) -> dict:
    """Upsert this principal's presence on the doc. Refreshes expires_at.

    Requires the principal to have view access (enforced at a higher layer —
    here we only verify the doc exists; callers that bypass the permission
    layer get a weaker check). The spec notes the server does not require
    *edit* access to set status='editing'; presence is advisory.

    Returns `{doc_id, status, expires_at}` for the caller to echo back.
    """
    if status not in _VALID_STATUSES:
        raise ValueError(f"status must be one of {_VALID_STATUSES!r}, got {status!r}")
    if principal.principal_type not in ("user", "agent"):
        raise ValueError(f"principal.principal_type must be 'user' or 'agent', got {principal.principal_type!r}")
    if not _doc_exists(conn, doc_id):
        raise PresenceError(f"document {doc_id} not found")

    ts = _now(now)
    updated_at = ts.isoformat(timespec="seconds")
    expires_at = (ts + _TTL).isoformat(timespec="seconds")

    conn.execute(
        """
        INSERT INTO presence (doc_id, principal_id, principal_type, status, note, updated_at, expires_at)
        VALUES (?, ?, ?, ?, ?, ?, ?)
        ON CONFLICT(doc_id, principal_id) DO UPDATE SET
            principal_type = excluded.principal_type,
            status         = excluded.status,
            note           = excluded.note,
            updated_at     = excluded.updated_at,
            expires_at     = excluded.expires_at
        """,
        (doc_id, principal.principal_id, principal.principal_type, status, note, updated_at, expires_at),
    )
    conn.commit()
    return {"doc_id": doc_id, "status": status, "expires_at": expires_at}
```

```python
def clear_status(
    conn: sqlite3.Connection,
    *,
    doc_id: str,
    principal: Principal,
) -> dict:
    """Remove this principal's presence row. Idempotent."""
    conn.execute(
        "DELETE FROM presence WHERE doc_id = ? AND principal_id = ?",
        (doc_id, principal.principal_id),
    )
    conn.commit()
    return {"ok": True}


def list_active(
    conn: sqlite3.Connection,
    *,
    doc_id: str,
    now: datetime | None = None,
) -> list[ActivePrincipal]:
    """Return non-expired presence rows for the doc, with display_name joined.

    The `expires_at > ?` filter is applied in SQL so stale rows do not leak
    even if the background GC is behind or has crashed.
    """
    ts = _now(now).isoformat(timespec="seconds")
    rows = conn.execute(
        """
        SELECT
            p.principal_id   AS principal_id,
            p.principal_type AS principal_type,
            CASE
                WHEN p.principal_type = 'user'  THEN u.display_name
                WHEN p.principal_type = 'agent' THEN a.display_name
                ELSE NULL
            END              AS display_name,
            p.status         AS status,
            p.note           AS note,
            p.updated_at     AS updated_at
        FROM presence p
        LEFT JOIN users  u ON p.principal_type = 'user'  AND u.id = p.principal_id
        LEFT JOIN agents a ON p.principal_type = 'agent' AND a.id = p.principal_id
        WHERE p.doc_id = ?
          AND p.expires_at > ?
        ORDER BY p.updated_at DESC
        """,
        (doc_id, ts),
    ).fetchall()
    return [
        ActivePrincipal(
            principal_id=r["principal_id"],
            principal_type=r["principal_type"],
            display_name=r["display_name"],
            status=r["status"],
            note=r["note"],
            updated_at=r["updated_at"],
        )
        for r in rows
    ]
```

```python
def gc_expired(
    conn: sqlite3.Connection,
    *,
    now: datetime | None = None,
) -> int:
    """Delete rows where expires_at <= now. Returns the count deleted.

    Safe to call from a background task concurrently with reads because
    `list_active` filters on `expires_at > now` independently. GC is a storage
    hygiene job, not a correctness mechanism.
```

```python
"""Background asyncio task that sweeps expired presence rows every N seconds.

Registered on the FastAPI app's lifespan. Self-contained — pulls its own
DB connection from the app state at call time. Failures are logged and
swallowed; the loop continues ticking.
"""

from __future__ import annotations

import asyncio
import logging
import sqlite3
from typing import Callable

from markland.service import presence

logger = logging.getLogger("markland.presence.gc")


async def _loop(
    gc_callable: Callable[..., int],
    *,
    interval_seconds: float,
    stop_event: asyncio.Event,
) -> None:
    """Call `gc_callable()` every `interval_seconds` until `stop_event` is set.

    Factored out of `start()` so unit tests can exercise the loop without a
    full FastAPI app or DB.
    """
    while not stop_event.is_set():
        try:
            deleted = gc_callable()
            if deleted:
                logger.info("presence gc deleted %d expired rows", deleted)
        except Exception:  # pragma: no cover - defensive; tested via injected bad callable
            logger.exception("presence gc tick failed; continuing")
        try:
            await asyncio.wait_for(stop_event.wait(), timeout=interval_seconds)
        except asyncio.TimeoutError:
            pass


def start(
    conn: sqlite3.Connection,
    *,
    interval_seconds: float = 60.0,
) -> tuple[asyncio.Task, asyncio.Event]:
    """Start the GC task. Returns (task, stop_event) for lifespan teardown."""
    stop_event = asyncio.Event()

    def _gc_once() -> int:
        return presence.gc_expired(conn)

    task = asyncio.create_task(_loop(_gc_once, interval_seconds=interval_seconds, stop_event=stop_event))
    return task, stop_event


async def stop(task: asyncio.Task, stop_event: asyncio.Event) -> None:
    """Signal the loop to stop, then wait for it to exit."""
    stop_event.set()
    try:
        await asyncio.wait_for(task, timeout=5.0)
    except asyncio.TimeoutError:
        task.cancel()
        try:
            await task
        except (asyncio.CancelledError, Exception):
            pass
```

```python
"""HTTP API for presence.

All three endpoints require an authenticated principal (session cookie for
web callers or bearer token for MCP — both resolve to a Principal via the
dependencies established in plan 2). Presence is a per-principal operation;
there is no "admin sets presence on behalf of X" path.
"""

from __future__ import annotations

import sqlite3
from typing import Literal

from fastapi import APIRouter, Depends, HTTPException
from pydantic import BaseModel, Field

from markland.service import presence
from markland.web.deps import require_principal_http  # plan 2: session/bearer → Principal


class _PresenceBody(BaseModel):
    status: Literal["reading", "editing"]
    note: str | None = Field(default=None, max_length=500)


def create_presence_router(db_conn: sqlite3.Connection) -> APIRouter:
    router = APIRouter(prefix="/api/docs/{doc_id}/presence", tags=["presence"])

    @router.post("")
    def set_presence(doc_id: str, body: _PresenceBody, principal=Depends(require_principal_http)):
        try:
            return presence.set_status(
                db_conn,
                doc_id=doc_id,
                principal=principal,
                status=body.status,
                note=body.note,
            )
        except presence.PresenceError as exc:
            raise HTTPException(status_code=404, detail=str(exc))
        except ValueError as exc:
            raise HTTPException(status_code=400, detail=str(exc))

    @router.delete("")
    def clear_presence(doc_id: str, principal=Depends(require_principal_http)):
        return presence.clear_status(db_conn, doc_id=doc_id, principal=principal)

    @router.get("")
    def list_presence(doc_id: str, principal=Depends(require_principal_http)):
        # Permission: the principal must have view access. We piggyback on
        # service.docs.get's permission check by calling it and discarding.
        from markland.service import docs as _docs
        _docs.get(db_conn, doc_id, principal)  # raises NotFound/Forbidden as appropriate
        actives = presence.list_active(db_conn, doc_id=doc_id)
        return [
            {
                "principal_id": a.principal_id,
                "principal_type": a.principal_type,
                "display_name": a.display_name,
                "status": a.status,
                "note": a.note,
                "updated_at": a.updated_at,
            }
            for a in actives
        ]

    return router
```

```python
"""FastAPI dependencies that resolve the caller's principal."""

from fastapi import Cookie, Header, HTTPException

from markland.service import auth as _auth


def require_principal_http(
    markland_session: str | None = Cookie(default=None),
    authorization: str | None = Header(default=None),
):
    p = None
    if markland_session:
        p = _auth.principal_from_session(markland_session)
    if p is None and authorization and authorization.lower().startswith("bearer "):
        p = _auth.principal_from_token(authorization[7:].strip())
    if p is None:
        raise HTTPException(status_code=401, detail="unauthenticated")
    return p
```

## What this plan does NOT deliver

Per the spec §9.3, §15, and §17, the following are explicitly out of scope:

- **No live updates** — the web viewer requires a page refresh to reflect presence changes; there is no SSE stream, no WebSocket, no polling on the client.
- **No cursors or selection ranges** — presence is a coarse "reading vs editing" flag per doc, not per-section.
- **No "done" / "finished" status** — completing work means `clear_status` or let expire. The state machine is `absent | reading | editing`.
- **No enforcement** — a principal marked `editing` does not block others from writing. Conflict safety comes from plan 8's version / `if_version` surface, not from presence.
- **No per-principal rate limits on `set_status`** — rate limiting lands in plan 10.
- **No audit-log rows for presence changes** — high churn, low signal; deferred.
- **No email or push notification when someone starts editing a doc** — noisy; deferred.
- **No presence-history surface ("who was here yesterday")** — the spec keeps state space tight; deferred until a real need emerges.
- **No cross-doc "active now" dashboard** — presence surfaces only per-doc via `markland_get` and `/d/{share_token}`.
- **No migration of presence state** — fresh DB on first deploy; presence table starts empty and is authoritative.
